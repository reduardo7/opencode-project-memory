# Session 2026-05-07

## Context

User pointed out that sub-agents spawned via `Agent(subagent_type: "general-purpose", prompt: ...)` don't respect the `model:` field in the SKILL.md frontmatter. The SKILL.md `model:` field only applies when the skill is invoked via the `Skill` tool.

## Decisions

- `memory-digest/SKILL.md` must explicitly pass `model: sonnet` when spawning both `memory-digest-daily` and `memory-digest-spec` sub-agents
- The `model` parameter in the `Agent` tool call is the only way to set the model for `general-purpose` sub-agents
- `memory-search` is invoked via `Skill` tool → its `model: haiku` frontmatter is respected automatically

## Learnings

- `Agent(subagent_type: "general-purpose", model: "sonnet", ...)` — model must be passed explicitly
- `Skill(skill: "plugin:name")` — respects the `model:` field in the SKILL.md frontmatter
- SKILL.md frontmatter (`model:`, `allowed-tools:`) is only applied when the skill is invoked as a skill, not when its content is injected into an Agent prompt

## References

- `skills/memory-digest/SKILL.md` — added `model: sonnet` to both Agent spawn instructions and sub-agents table
- `docs/vault/Claude/Memory.md` — updated flow diagram, sub-agent section headings and descriptions
