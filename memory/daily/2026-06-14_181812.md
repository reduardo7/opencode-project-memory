- Hook mapping (7 Python hooks â†’ 1 JS plugin `.opencode/plugin/memory.js`):

  - PreCompact â†’ `experimental.session.compacting` (push reminder into `output.context`).

  - Stop â†’ `event` hook on `session.idle`; surface a TUI toast (OpenCode can't re-inject conversation context post-stop; the always-on system.transform reminder is the primary net).

## Learnings
- OpenCode is highly Claude-compatible: natively reads `.claude/skills/`, `CLAUDE.md` (fallback), and the SKILL.md format matches.
- OpenCode plugin context object: `{ project, client, $, directory, worktree }`. Plugins export an async fn returning a hooks object. Custom tools via `tool()` + Zod.
- `experimental.chat.system.transform(input,output)` â†’ `output.system.push(str)` injects into the system prompt; `input` has `sessionID`, `model` (NOT the user message yet â€” so no per-prompt keyword heuristics possible).
- `experimental.session.compacting(input,output)` â†’ `output.context.push(str)` or `output.prompt = str`.
- Experimental hook field names vary across versions â†’ plugin must be defensive (optional chaining, try/catch, array guards).
